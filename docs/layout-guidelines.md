# Human Layout Revision Guidelines

These rules come from repeated manual corrections of AI-generated slide drafts.

## Text Spacing

Use paragraph line spacing around `1.25`. Values like `0.8` or `1.0` may cause dense Chinese text to visually collapse, especially inside compact PPT text boxes.

## Line Wrapping

Keep complete phrases on one line when space allows. Avoid leaving only one or two Chinese characters on the next line.

If a line wraps badly, prefer:

- slightly reducing font size
- widening the text box
- moving the phrase to a new bullet
- rewriting the sentence more compactly

## Consistency

Content at the same level should use the same visual rules:

- same font size
- same font weight
- same text color
- same border radius
- same icon size
- same card padding
- same spacing between repeated elements

## Layout Density

Try to use the whole slide area. Avoid leaving too much empty space at the bottom.

Good layout is not simply "more content". It should feel full, aligned, and breathable.

## Alignment

Prioritize clear edges:

- align columns by their top and bottom edges
- align cards within the same row
- keep footer and page number positions stable
- use consistent left margins across slides

## Tables

Tables are high-risk in 16:9 slides. Before exporting:

- check all rows are visible
- reduce padding before reducing font size
- avoid tall paragraph text inside table cells
- use compact labels where possible

## Images

Images should be placed where they support the argument, not as decoration.

Check:

- no stretching
- important labels remain readable
- image captions do not overlap
- margins are balanced
